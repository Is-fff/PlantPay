# 工厂货物产量与工资结算系统 — 需求文档

## 一、项目背景

工厂老板需要一个移动端工具，用于：
- 记录员工每天的货物生产量（区分颜色、码数）
- 月底自动计算每位员工的工资

当前为人工记账方式，效率低、易出错。

## 二、用户画像

- **唯一用户**：老板（单人使用，无需登录/多账号）
- **使用场景**：在工厂车间用手机随时记录产量，月底结算工资

## 三、技术方案

| 决策项 | 选择 |
|--------|------|
| 运行平台 | 移动端 Web App（浏览器访问，响应式设计） |
| 数据存储 | 浏览器本地存储（IndexedDB），无需后端服务 |
| 是否需要联网 | 不需要，纯离线可用 |

> PWA 可作为后续增强项（添加到主屏幕、离线缓存等），初期先做普通 Web App。

## 四、核心概念 / 数据实体

### 1. 员工（Employee）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | 唯一标识 | 自动生成 |
| name | 文本 | 员工姓名 |
| remark | 文本 | 备注（可选） |

### 2. 型号 / 货物（Product）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | 唯一标识 | 自动生成 |
| name | 文本 | 型号名称，如 "M01" |
| unitPrice | 数字 | 单价（元/件），同型号所有颜色码数单价一致 |
| remark | 文本 | 备注（可选） |

### 3. 颜色（Color）— 按型号配置
每个型号可配置自己独立的颜色列表。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | 唯一标识 | 自动生成 |
| productId | 关联 | 属于哪个型号 |
| name | 文本 | 颜色名称，如 "红"、"蓝" |

### 4. 码数 / 尺码（Size）— 按型号配置
每个型号可配置自己独立的码数列表。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | 唯一标识 | 自动生成 |
| productId | 关联 | 属于哪个型号 |
| name | 文本 | 码数名称，如 "S"、"M"、"L"、"XL" |

### 5. 产量记录（ProductionRecord）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | 唯一标识 | 自动生成 |
| employeeId | 关联 | 哪位员工生产的 |
| productId | 关联 | 哪种型号 |
| colorId | 关联 | 什么颜色 |
| sizeId | 关联 | 什么码数 |
| quantity | 整数 | 合格品数量（件） |
| defectiveQuantity | 整数 | 不合格品数量，可空/默认0 |
| date | 日期 | 生产日期 |
| createdAt | 时间戳 | 记录创建时间 |

> **去重规则**：同一员工 + 同一型号 + 同一颜色 + 同一码数 + 同一天 → 累加到同一条记录（数量叠加）。

### 6. 工资结算（结算视图，不单独存储）
由产量记录实时计算得出：

```
某员工当月工资 = Σ ( (合格数量 - 不合格数量) × 对应型号单价 )
```

不合格品数量为可选项，未填写时默认为 0，不扣除。

## 五、功能模块

### 模块 1：货物型号管理
- 型号列表查看
- 新增型号（名称、单价）
- 编辑型号信息
- 删除型号（已被引用的不可删除，需二次确认）
- 每个型号内管理自己的颜色列表（增删改查）
- 每个型号内管理自己的码数列表（增删改查）

### 模块 2：员工管理
- 员工列表查看
- 新增员工（姓名）
- 编辑员工信息
- 删除员工（已被引用的不可删除，需二次确认）

### 模块 3：产量记录
- **记录页面**（核心页面）：选择日期 → 选择员工 → 选择型号（自动带出单价）→ 选择颜色 → 选择码数 → 输入数量 → 保存
- **快速记录模式**：日期默认当天，可连续为同一员工录入多条记录
- **当天记录列表**：查看今日已录记录，支持删除
- **历史记录查询**：按日期范围、员工筛选查看

### 模块 4：工资结算
- 选择月份 → 查看所有员工当月工资汇总
- 汇总展示：
  - 每个员工当月总产量（件数）
  - 每个员工当月总工资（元）
- 点击员工可查看明细：
  - 按型号分组，显示每件的数量、单价、小计
  - 按日期排列的详细记录

### 模块 5：数据安全
- 数据导出（JSON 备份文件下载）
- 数据导入（JSON 备份文件恢复）
- 清空数据（需二次确认 + 密码确认）

## 六、页面结构

```
首页（导航中心）
├── 📦 货物管理
│   └── 型号列表 → 型号详情 → 管理该型号的颜色和码数
├── 👥 员工管理
│   └── 员工列表 → 新增/编辑 员工
├── 📝 记录产量
│   ├── 新增记录表单（选择型号后联动加载该型号的颜色和码数）
│   └── 今日记录 / 历史记录
├── 💰 工资结算
│   ├── 选择月份
│   ├── 员工工资汇总表
│   └── 员工工资明细
└── ⚙️ 设置
    └── 数据导入/导出
```

## 七、界面设计要求

- 移动端优先，适配 375px–414px 宽度（主流手机屏幕）
- 大按钮、大字体，方便车间操作
- 表单尽量使用选择器（下拉/底部弹出），减少手动输入
- 数字输入使用数字键盘
- 表格在手机上可用卡片布局代替

## 八、非功能性需求

- 数据全部存本地 IndexedDB，无后端依赖
- 页面加载 < 2 秒
- 数据量预估：单工厂 < 20 员工，年记录 < 5000 条，无性能瓶颈
- 首次使用时需要初始化引导（预置颜色、码数等基础数据）

## 九、需求确认结论

| 问题 | 结论 |
|------|------|
| 颜色和码数范围 | **按型号不同**：每个型号独立配置颜色和码数 |
| 重复录入处理 | **累加**：同一人同一天同型号同颜色同码数，数量自动累加 |
| 不合格品 | 每条记录可**选填**不合格品数量，有值则从工资中扣除 |
